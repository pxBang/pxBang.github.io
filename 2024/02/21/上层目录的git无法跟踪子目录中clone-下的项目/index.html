上层目录的git无法跟踪子目录中clone 下的项目

2.目录树形图

.
└── A
    ├── B
    │   └── .git
    │       ├── branches
    │       ├── hooks
    │       ├── info
    │       ├── objects
    │       │   ├── info
    │       │   └── pack
    │       └── refs
    │           ├── heads
    │           └── tags
    └── .git
        ├── branches
        ├── hooks
        ├── info
        ├── objects
        │   ├── info
        │   └── pack
        └── refs
            ├── heads
            └── tags

3.解法

  1. 想办法删B文件夹下的.git文件夹
    • git rm -r path
  2. 清空A文件夹git缓存中B文件夹，再手动添加B文件夹
    • git rm -r --cached path
    • git add path

解法1，2均可
